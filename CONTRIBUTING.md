## Linee guida (brevi)

- Usa **GitHub Discussions** per domande e chiarimenti.
- Apri una **Issue** solo per bug riproducibili o miglioramenti concreti.
- Mantieni le proposte **minimali** e coerenti con gli obiettivi didattici del corso.
- Evita di aggiungere dataset reali o file pesanti nel repository.

## Workflow

- Crea una branch descrittiva.
- Commits piccoli e con messaggi chiari.
- Apri una Pull Request con descrizione del cambiamento e motivazione.
